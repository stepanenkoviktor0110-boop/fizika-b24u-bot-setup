# Fizika Feed Proxy

Перестраивает YML-фид Domoplaner так, чтобы карточки квартир выигрывали в семантическом поиске B24U RAG. Проблема: длинные PDF (4000+ знаков) и общие описания ЖК вытесняют короткие карточки фида (~100 знаков) из топ-5 источников. Бот не видит реальных URL квартир и галлюцинирует короткие токены `l.b24u.ru/<токен>`.

Решение — обогатить `<description>` каждого `<offer>` синонимами комнат, ЖК, корпуса, секции, цены, отделки и адреса. Текст карточки растёт до ~400-500 знаков и начинает попадать в топ RAG.

## Что делает скрипт

`scripts/build-feed.mjs`:

1. Тянет исходный YML с domoplaner.ru (URL зашит в скрипте).
2. Парсит, для каждого offer извлекает `<rooms>`, `<building-name>`, `<building-section>`, `<floor>`, `<price>`, `<renovation>`, исходный `<description>`, `<adress>`.
3. Строит обогащённое описание (см. ниже), кладёт в `<description>`. Остальные поля — без изменений.
4. Восстанавливает атрибут `available="true"` (fast-xml-parser сворачивает boolean-like значения).
5. Пишет `public/feed.xml`.

## Что попадает в `<description>`

- **Комнаты:** все синонимы из RAG-словаря B24U: `1-комнатная, однокомнатная, однушка, 1к, 1-комн, 1-к` (для `<rooms>=1`); пентхаус для 5+. Для `<rooms>=студия` — `студия, квартира-студия, студийная квартира`.
- **ЖК:** все варианты названия: `Talento, Таленто`, `VIDI, Vidi, Види, ВиДи`, `Остров Первых, Остров, Матисов остров`, `Моисеенко 10, Моисеенко, ул. Моисеенко 10`.
- **Корпус:** выводится из `<building-section>` по таблице маппинга:
    - **Моисеенко 10**: секция 1→К1, 2/3→К2, 4→К3, 5→К4.
    - **Talento**: всегда корпус 1 (один корпус, 2 секции).
    - **Остров Первых**: формат поля `«N корпус M секция»` — корпус N парсится напрямую.
    - **VIDI**: структура не загружена, корпус не упоминаем.
- **Секция:** короткая форма `с.N` плюс полная.
- **Этаж, площадь, цена, отделка, адрес** — естественным языком.
- **Оригинальный description** дописывается в конце как fallback.

## Запуск локально

```bash
cd proxy-feed
npm install
LOCAL=1 npm run build   # читает old/artifacts/domoplaner-feed-original-2026-05-09.yml
npm run build           # читает живой URL Domoplaner
SKIP_URL_CHECK=1 npm run build  # без HTTP-проверки (быстрее, но не отсеет проданные)
```

Output: `../public/feed.xml`.

## Live URL liveness check

Скрипт логинится в `booking.fizika.group` под партнёрской учёткой и для каждого `<url>` проверяет, отдаёт ли портал реальную страницу объекта. Признаки live:
- Path после redirect — `/flat/<id>/` или `/commerce/<id>/`.
- Тело > 10 000 символов (живые flat ≈ 66KB, commerce ≈ 37KB; 404/снято ≈ 0).

Без credentials (`BOOKING_FIZIKA_LOGIN`/`BOOKING_FIZIKA_PASSWORD`) портал отдаёт всё на `/auth/`, поэтому проверка пропускается с warning'ом — фид публикуется без фильтрации.

**Локально:** креды лежат в `.env` корня репо. **В CI:** добавить как GitHub Secrets с теми же именами.

## Деплой через GitHub Pages

Workflow `.github/workflows/build-feed.yml` крутит cron `0 */6 * * *` (каждые 6 часов) и публикует `public/feed.xml` в ветку `gh-pages` через `peaceiris/actions-gh-pages`.

**Один раз вручную после первого push:**

1. `Settings → Secrets and variables → Actions → New repository secret` — добавить:
   - `BOOKING_FIZIKA_LOGIN` — телефон партнёра (с `+7`).
   - `BOOKING_FIZIKA_PASSWORD` — пароль.
2. `Actions → Build proxy feed → Run workflow` — первый прогон создаст ветку `gh-pages`.
3. `Settings → Pages → Source: Deploy from a branch → Branch: gh-pages / (root)`.
4. Через ~1 минуту после Save фид доступен по URL:

```
https://stepanenkoviktor0110-boop.github.io/<repo>/feed.xml
```

## Подключение в B24U

После того, как Pages-URL заработает:

1. Партнёрский кабинет → `client_id=643000000002760` → вкладка «База знаний» → «Лента».
2. **Добавить новый фид** (НЕ заменять существующий): поле URL = Pages-URL, тип `yml`, sync 12h.
3. Через 12-24 часа после первого синка — RAG-тест на «1-комнатные Остров Первых до 25 млн». Карточки фида должны появиться в топ-5.

## Откат

- В B24U — отключить proxy-фид в «Лента», оставить только оригинал Domoplaner.
- Локальный снимок исходного фида: `old/artifacts/domoplaner-feed-original-2026-05-09.yml`.
